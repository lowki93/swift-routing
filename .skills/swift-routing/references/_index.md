## Core References

Recommended order:
1. `routes.md`: Define route enums and route-to-view mappings.
2. `routing-view.md`: Create navigation stacks and understand router ownership.
3. `router.md`: Use `Router` for push, present, cover, back, and root updates.

Then by topic:

- `deeplinks.md`: Convert external links into navigable routes.

- `navigation-link.md`: Use `NavigationLink` with route-driven navigation.
- `troubleshooting.md`: Diagnose lifecycle, routing, and state edge cases.
